# 坎尼战役 - 实现文档

> **公元前216年 | 罗马 vs 迦太基**
> 汉尼拔最完美的包围战，军事史教科书

---

## 战役信息

### 基本信息
- **时间**: 公元前216年8月2日
- **地点**: 意大利坎尼（阿普利亚地区）
- **交战方**: 罗马共和国(约86,000人) vs 迦太基(约40,000-50,000人)
- **伤亡**: 罗马约50,000-70,000人 vs 迦太基约5,000-8,000人
- **统帅**: 瓦罗(罗马) vs 汉尼拔·巴卡(迦太基)

### 战术亮点
- 军事史上最完美的包围战
- 故意示弱，中央后退形成月牙形
- 两翼骑兵击败罗马骑兵后回头包抄
- 双重包围彻底歼灭罗马大军

### 战斗阶段

**阶段0: 对峙 (0-10秒)**
- 双方列阵
- 罗马军队人数众多，气势逼人
- 汉尼拔阵型看似薄弱
- 相机展示双方规模对比

**阶段1: 接触 (10-30秒)**
- 罗马方阵缓慢推进
- 迦太基中央高卢/伊比利亚步兵缓慢后退
- 罗马中央被诱入"口袋"
- 相机跟随罗马推进

**阶段2: 骑兵交战 (30-50秒)**
- 两翼骑兵交战
- 努米底亚骑兵击败罗马左翼骑兵
- 迦太基重骑兵击败罗马右翼骑兵
- 罗马骑兵被逐出战场

**阶段3: 包围开始 (50-80秒)**
- 迦太基两翼的非洲步兵转向
- 骑兵从后方返回
- 罗马军队被三面包围
- 相机高空俯视，展示包围态势

**阶段4: 包围完成 (80-100秒)**
- 双重包围完成
- 罗马军队无法逃脱
- 密集方阵变成混乱绞肉机
- 罗马士兵成片倒下

**阶段5: 屠杀 (100-120秒)**
- 罗马军队被逐步压缩
- 密集方阵互相挤压
- 迦太基士兵从外围屠杀
- 战场变成血腥绞肉机

**阶段6: 结束 (120秒+)**
- 幸存者寥寥
- 迦太基胜利
- 展示战场惨状
- 战役信息统计

## 技术实现要点

### 动态阵型系统

**月牙形阵型**:
```javascript
// 迦太基中央后退逻辑
中央单位位置 = 基础位置 + (时间 × 后退速度)
保持弧形阵型
```

**非洲步兵转向**:
```javascript
// 阶段3触发
if (阶段3开始) {
  非洲步兵角度 += 90度; // 面向罗马侧翼
  开始包抄移动;
}
```

**骑兵返回**:
```javascript
// 骑兵击败罗马骑兵后
骑兵路径 = [
  击败罗马骑兵位置,
  战场边缘,
  罗马军队后方,
  转向包围
];
```

### 相机角度

| 阶段 | 相机位置 | 说明 |
|------|----------|------|
| 对峙 | 高空俯视 | 双方规模对比 |
| 接触 | 中距离跟随 | 罗马推进视角 |
| 骑兵交战 | 侧视平视 | 骑兵冲锋 |
| 包围开始 | 高空俯视 | 包围态势形成 |

## 优化建议

### 性能优化
- 单位数量控制在500-800
- GPU实例化相同单位
- 距离剔除远处单位
- 简化碰撞检测

### 战术优化
- 阵型变化要平滑
- 包围过程要清晰可见
- 战斗节奏要合理

### 可扩展性
- 包围战术系统可复用
- 罗马方阵可复用到其他战役
- 骑兵系统可复用
