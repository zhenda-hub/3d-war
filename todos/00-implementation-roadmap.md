# 战役实现路线图

> 基于 todo2.md 精选的4个战役，按实现难度从简单到复杂排列

---

## 实现顺序

### 1. 马拉松战役 ⭐ 原型验证
**难度**: ★★☆☆☆
**时间**: 约16天
**文件**: `01-marathon-battle.md`

**选择理由**:
- 最简单的平原地形
- 仅2种兵种（希腊重步兵 vs 波斯弓箭手）
- 单位数量少（200-300单位）
- 战术相对简单（方阵冲锋+双翼包抄）
- 适合作为技术验证和框架搭建

**核心挑战**:
- 方阵冲锋动画
- 基础战斗系统
- 简单AI行为

**可复用成果**:
- 地形系统
- 单位模型框架
- 基础战斗系统
- 阵型排列系统

---

### 2. 克雷西战役 ⭐ 箭雨展示
**难度**: ★★★☆☆
**时间**: 约21天
**文件**: `02-crecy-battle.md`

**选择理由**:
- 在马拉松基础上增加箭雨系统
- 高地地形+防御工事
- 3种兵种（长弓手、重骑士、十字弓手）
- 视觉震撼（箭雨效果）

**核心挑战**:
- 箭雨系统（弹道、抛物线、命中）
- 骑兵冲锋动画
- 多波次调度
- 视觉特效优化

**可复用成果**:
- 箭雨系统（可复用到阿金库尔）
- 骑兵系统
- 防御工事系统

---

### 3. 阿金库尔战役 ⭐ 泥泞创新
**难度**: ★★★☆☆
**时间**: 约25天
**文件**: `04-agenincourt-battle.md`

**选择理由**:
- 在克雷西基础上增加泥泞系统
- 独特的视觉体验（泥泞战场）
- 狭窄战场限制
- 情感冲击力强

**核心挑战**:
- 泥泞物理系统
- 泥浆飞溅粒子效果
- 骑兵与泥泞的交互
- 天气效果（阴沉）

**可复用成果**:
- 泥泞系统
- 天气系统
- 复杂地形系统

---

### 4. 坎尼战役 ⭐ 战术巅峰
**难度**: ★★★★☆
**时间**: 约24天
**文件**: `03-cannae-battle.md`

**选择理由**:
- 军事史上最经典的包围战
- 动态阵型变化（月牙形后退）
- 双重包围战术
- 大规模单位（500-800单位）

**核心挑战**:
- 动态阵型系统
- 复杂战术AI
- 大规模单位管理
- 多阶段战斗协调

**可复用成果**:
- 动态阵型系统
- 包围战术AI
- 大规模单位管理

## 技术栈建议

### 核心技术
- **Three.js / React Three Fiber** - 3D渲染
- **React** - UI和状态管理
- **TypeScript** - 类型安全
- **Vite** - 构建工具

### 辅助库
- **@react-three/drei** - Three.js辅助组件
- **@react-three/fiber** - React渲染器
- **zustand** - 轻量状态管理
- **gsap** - 动画库（可选）

### 性能优化
- **GPU Instancing** - 单位实例化渲染
- **LOD System** - 距离级别细节
- **Object Pooling** - 对象池复用
